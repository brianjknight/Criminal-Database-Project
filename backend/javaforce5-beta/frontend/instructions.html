Criminal Records Database Homepage

Instructions

This is a database of criminal records consisting of 2 tables; Criminal Records & Crimes. Two tables were used for the scope of this project for purposes of interacting with multiple entities. The service is built using AWS Lambda and API Gateway to perform CRUD operations on the DynamoDB tables.

For the scope of this project, the Crime Manager page is not functional. Below you can find the dataset for the Criminal Records and Crimes tables. Note that in order to add a crime to a criminal record, the crime must exist with a matching SSN. To test, expunge(delete) a criminal record, create a new record, then add crime(s) with that SSN to the criminal record.

The Criminal Records Filter page currently has only one filter available. It uses a global secondary index (GSI) on the Criminal Records table with optional min and max number corresponding to the crime count field.
